АВТОМАТИЗАЦИЯ ТЕСТИРОВАНИЯ API DADATA
    Проект представляет собой набор автотестов для проверки REST API сервиса "Дадата".
    Тесты покрывают как позитивные сценарии использования, так и негативные (проверка безопасности, обработка некорректных данных и граничных значений).

ТЕХНОЛОГИЧЕСКИЙ СТЕК
    Язык программирования: Java 17
    Фреймворк для тестирования: JUnit 5 (Jupiter)
    HTTP-клиент: REST Assured
    Обработка JSON: Jackson
    Сборщик: Maven

ПОКРЫТЫЕ МЕТОДЫ (ENDPOINTS)
    В рамках проекта протестированы 3 основных метода API:
    POST /suggest/address — подсказки по адресам;
    GET /iplocate/address — определение города по IP-адресу;
    POST /suggest/fio — подсказки по ФИО (определение пола и разбивка строки).

ОСОБЕННОСТИ РЕАЛИЗАЦИИ

- Использован паттерн DTO: запросы и ответы строго типизированы с помощью POJO-классов (Request/Response модели).
- Использованы спецификации REST Assured, выделены базовые спецификации запросов и ответов для устранения дублирования кода.
- Использование CSV-данных для прогона сценариев с различными наборами входных параметров.
- Настроен конкурентный запуск тестов средствами JUnit 5 (параллельное выполнение).
- Использована аннотация ResourceLock для управления пулом потоков и предотвращения сброса SSL-соединения со стороны защиты QRATOR на тарифе Free.

ИНСТРУКЦИЯ ПО ЗАПУСКУ

Настройка окружения
Для успешного прохождения тестов необходимо передать авторизационный токен DaData.
В вашей операционной системе или в конфигурации запуска среды разработки добавьте переменную окружения DADATA_TOKEN

Запуск через Maven
Для прогона всех тестов (включая параллельное выполнение) откройте терминал в корне проекта и выполните команду:
mvn clean test

Структура проекта

src/test/java/api_dadata/config — Конфигурационные файлы.
src/test/java/api_dadata/dto — Модели данных для десериализации JSON.
src/test/java/api_dadata/service — API-клиенты для отправки запросов.
src/test/java/api_dadata/spec — Базовые настройки REST Assured.
src/test/java/api_dadata/test — Исполняемые тестовые классы с логикой проверок.
